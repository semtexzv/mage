# SDK Crate Structure

The `mage` crate (`pkg/sdk`) is the single dependency for extension authors
and the top-level namespace for the host binary. It re-exports the full
module tree — no feature gates, no optional deps. Every build has the same
capabilities.


## Module Tree

```rust
// pkg/sdk/src/lib.rs

/// Extension trait, hooks, dispositions, amendment types, factory registry.
pub use mage_core::extension;

/// Tool trait, execution tiers, results, mailbox.
pub use mage_core::tool;

/// Agent message types, events, delivery modes.
pub use mage_core::types;

/// LLM abstraction: Provider trait, Message, Model, CancelToken, events.
pub use mage_llm as llm;

/// Ref-counted strings: Str, Str<Atomic>.
pub use refstr;

/// Terminal UI: renderer, markdown, editor, keymap, styles.
pub use mage_tui as tui;

/// Dynamic workspace compiler.
pub use mage_build as build;

```


## Prelude

`use mage::prelude::*` gives extension authors everything needed to write
a tool or extension hook.

```rust
pub mod prelude {
    // Extension system
    pub use crate::extension::{
        Extension, Registry, FactoryRegistry, HookCtx, HookFuture,
        Disposition, BeforeStartAmend, ToolResultAmend, InputAmend,
        ContextAmend, CompactAmend, BashAmend,
        ToolCallArgs, ToolResultArgs, BeforeStartArgs, TurnEndArgs,
        MessageArgs, MessageDeltaArgs, ToolExecStartArgs, ToolExecEndArgs,
        BeforeForkArgs, UserBashArgs, AgentEndArgs, ModelSelectArgs,
    };

    // Tools
    pub use crate::tool::{
        Tool, ToolExecution, ToolResult, ToolContent,
        Mailbox, MailboxSender,
    };

    // Agent types
    pub use crate::types::{AgentMessage, AgentEvent, DeliverAs};

    // LLM types (the subset extensions actually touch)
    pub use crate::llm::CancelToken;
    pub use crate::llm::types::{
        Message, UserContent, ContentBlock, Model, Usage,
    };

    // Strings
    pub use refstr::Str;

}
```


## Cargo.toml

```toml
[package]
name = "mage"
version = "0.1.0"
edition = "2024"
rust-version = "1.85"

[dependencies]
mage-core = { path = "../core" }
mage-llm = { path = "../llm" }
mage-tui = { path = "../tui" }
mage-build = { path = "../build" }
refstr = { path = "../refstr" }
```


## Host Binary
The host binary (`mage-host`) is **not** a static `src/main.rs`. It is
code-generated by mage-build at compile time: the `main()` function,
extension registration, and provider wiring are all synthesized from the
current set of discovered extensions and the Jinja2 template.

The generated code depends on `mage` (the SDK) plus whatever providers
and extensions are in scope. The exact Cargo.toml is written into the
staging directory by mage-build — it is not a file in the repo.


## Usage Patterns

Extension author (single-file module):
```rust
// @dep mage = { path = "..." }
use mage::prelude::*;
struct MyTool;
impl Tool for MyTool {
    fn name(&self) -> &str { "my_tool" }
    fn description(&self) -> &str { "Does a thing" }
    fn parameters(&self) -> &serde_json::Value { &MY_SCHEMA }
    fn execute(&self, _id: &str, params: Value, _cancel: CancelToken) -> ToolExecution {
        ToolExecution::running(async move {
            ToolResult::success("done")
        })
    }
}
pub fn init(reg: &mut Registry) {
    reg.tool(MyTool);
}
```

Extension author (explicit imports):
```rust
// @dep mage = { path = "..." }
use mage::extension::{Extension, Registry, HookCtx, Disposition};
use mage::tool::Tool;
use mage::llm::CancelToken;
use refstr::Str;  // also available as mage::refstr::Str
```

Host binary:
```rust
use mage::prelude::*;
use mage::tui::{App, run};
use mage::build::Bundle;
use mage::llm::Provider;
```


## Third-party Dependencies
The SDK does NOT re-export `serde_json`, `serde`, `tokio`, or `reqwest`.
All third-party crates are added by extensions via `@dep` annotations:
```rust
// @dep serde_json = "1"
// @dep serde = { version = "1", features = ["derive"] }
// @dep tokio = { version = "1", features = ["process", "fs"] }
// @dep reqwest = { version = "0.12", features = ["json"] }
```
These are resolved by mage-build during compilation. The versions are
included in the bundle hash via Cargo.lock (see DESIGN-REPRODUCIBLE-BUILDS.md).
`serde_json` is so universally needed that the built-in template includes
it as a dependency of every generated binary. Extension authors still add
`@dep serde_json = "1"` if they use it directly — mage-build deduplicates.


## Rationale

**No feature gates.** Every mage binary includes the full stack: core,
LLM, TUI, build system. Extensions compiled into the binary have access
to everything. This simplifies the dependency graph and avoids conditional
compilation bugs. The cost is compile time for extensions that don't use
TUI or build — acceptable because mage-build caches aggressively and these
crates only compile once.

**`pub use mage_llm as llm`, not `pub mod llm`.** Straight re-export.
`mage::llm::Provider`, `mage::llm::types::Model`. No wrapper, no
indirection. Same for `tui` and `build`.

**Single dependency for extensions.** Extension authors write
`@dep mage = { path = "..." }` and get everything. They don't need to know
mage-core vs mage-llm vs refstr. The internal crate boundaries are an
implementation detail.

**Prelude is opinionated.** It exports everything an extension author
typically needs. Extension authors who want precision use explicit imports
from submodules. Both work.
